01 / SCAFFOLDING

The Correspondence field

Tracking the same physical hand point as it moves through time.

Physical intuition

A point cloud frame is a bag of 3D positions sampled from the visible surface of a hand. Frame to frame, the indices in that bag are arbitrary — point #137 in frame 5 has nothing to do with point #137 in frame 6. To reason about motion we need to know which points correspond to the same physical patch of skin across time. That mapping is the correspondence field.

It is built once, offline, by walking depth + spatial neighborhoods: for each point in frame $t$, find the closest match in frame $t{+}1$ that minimizes a weighted sum of (a) 3D Euclidean distance and (b) raw depth gradient distance.

Math

$$\text{corr}_{t \to t{+}1}(i) \;=\; \underset{j \in \mathcal{N}_r(i)}{\arg\min}\;\Big[\;\|\,p_i^{(t)} - p_j^{(t{+}1)}\,\|_2 \;+\; \lambda_d \cdot \big|d_i^{(t)} - d_j^{(t{+}1)}\big|\;\Big]$$

where $\mathcal{N}_r(i)$ is the spatial neighborhood of radius $r$ around $p_i$ in the next frame, $d$ is the raw depth value at the projected pixel, and $\lambda_d$ weights depth-gradient agreement against pure 3D distance. Confidence is exponentially decayed with distance, so unreliable matches receive low weight.

Implementation

# nvidia_dataloader.py – built once, cached per clip
corr_target = build_correspondence(
    raw_depth, raw_points,
    radius=2,
    depth_weight=0.25,
    confidence_scale=16.0,
)
# corr_target[i] -> flat index in next frame, or -1
# corr_weight[i] -> confidence in [0, 1]

Two downstream uses, both optional:

  • Correspondence-guided sampling: chain matched points so index i in frame t = index i in frame t{+}1
  • Auxiliary mask: compute which sampled points have valid matches, gate per-point losses
Figure 1. Correspondence as a 1-to-1 mapping between point clouds. Confidence-weighted; unreliable matches (gray) are ignored.
What we learned

Correspondence is infrastructure, not a learning signal. In the variants we tested where it gated the auxiliary loss directly (contrastive, prediction, infonce), only ~10% of sampled points had valid matches — too sparse for stable gradients. The 80.29% baseline doesn't use correspondence at all in its loss.

02 / GEOMETRIC FEATURE

Bearing-Quaternion QCC rigidity

Per-point measure of how rigidly a local patch is rotating.

Physical intuition

For a perfectly rigid body, every point rotates around the same axis by the same angle from frame to frame. So if we describe each point by the direction from the hand's center to that point, those directions all rotate by the same quaternion. Patches whose neighbors all share the same per-point rotation are rigid; patches whose neighbors disagree are deforming. That single number — neighborhood agreement on rotation — is the QCC rigidity score.

Concretely: parametrize a point's bearing as the rotation that takes a fixed reference vector (north $= [0,1,0]$) onto the unit direction from the centroid to that point. Compute that bearing in frames $t$ and $t{+}1$, take the relative rotation, and then check whether spatial neighbors of point $i$ all came up with the same relative rotation.

Math · step by step

$$\textbf{1.}\quad c^{(t)} = \tfrac{1}{2}\big(\min_{i} p_i^{(t)} + \max_{i} p_i^{(t)}\big), \qquad d_i^{(t)} = \frac{p_i^{(t)} - c^{(t)}}{\|p_i^{(t)} - c^{(t)}\|}$$ $$\textbf{2.}\quad q_i^{(t)} \;=\; \mathrm{Quat}\!\left(\hat{\mathbf{y}} \to d_i^{(t)}\right) \;=\; \Big[\,\cos\tfrac{\theta}{2},\;\; \frac{\hat{\mathbf{y}} \times d_i^{(t)}}{\|\hat{\mathbf{y}} \times d_i^{(t)}\|}\sin\tfrac{\theta}{2}\,\Big]\;,\quad \theta = \arccos(d_{i,y}^{(t)})$$ $$\textbf{3.}\quad q^{\text{fwd}}_i \;=\; q_i^{(t{+}1)} \;\cdot\; \overline{q_i^{(t)}}$$ $$\textbf{4.}\quad \mathrm{inc}_i \;=\; \frac{1}{|\mathcal{N}(i)|}\sum_{j \in \mathcal{N}(i)} 2\,\arccos\!\big|\langle\, q^{\text{fwd}}_i,\; q^{\text{fwd}}_j\,\rangle\big|$$ $$\textbf{5.}\quad \mathrm{rigidity}_i \;=\; \exp\!\left(-\,\frac{\mathrm{inc}_i}{\mathrm{median}(\mathrm{inc})}\right) \;\in\; (0,1]$$

Step 4 is the heart of it: $2\arccos|\langle q_a, q_b\rangle|$ is the geodesic distance on the unit quaternion sphere $S^3$, modded out by the double-cover so $q$ and $-q$ count as the same rotation. A neighborhood of points all rotating identically gives $\mathrm{inc}_i \approx 0 \Rightarrow \mathrm{rigidity}_i \approx 1$. A neighborhood with chaotic rotations gives $\mathrm{inc}_i$ large $\Rightarrow \mathrm{rigidity}_i \to 0$.

Figure 2a. Each point's bearing direction $d_i$ from the centroid. The bearing quaternion $q_i$ is the rotation that takes north onto $d_i$.
Figure 2b. The signal: in a rigidly rotating patch all neighbors' $q^{\text{fwd}}$ vectors agree; in a deforming patch they scatter on $S^3$.

Implementation

# reqnn_motion.py · _compute_bearing_qcc
def _compute_bearing_qcc(points_4d, num_frames, knn_k=10):
    xyz       = points_4d[..., :3]
    centroids = (xyz.amax(2) + xyz.amin(2)) / 2
    dirs      = (xyz - centroids[:,:,None]).normalize(dim=-1)

    # Bearing quaternion: rotation from north onto dir
    bearing_q = north_to_dir_quat(dirs)               # (B, F, P, 4)
    q_fwd     = hamilton(bearing_q[:,1:], conj(bearing_q[:,:-1]))

    # Per-point inconsistency = mean geodesic dist to k-NN q_fwd
    inc       = geodesic_distance_to_knn(q_fwd, k=knn_k).mean(-1)
    rigidity  = (-inc / inc.median()).exp()           # (B, P) in (0,1]
    return rigidity
Role in the pipeline

The rigidity score is computed once per forward pass and used to modulate encoder features: encoded *= 1 + tanh(W · rigidity) with $W$ initialized to zero. In ablation, disabling this modulation costs ≈ 0 accuracy, suggesting rigidity is mostly carried as input to the encoder rather than as an explicit gating signal. The QCC machinery's main contribution flows through the loss in §03.

03 / THE LOSS THAT WORKS

Grounded Cycle Consistency

Predicting segment-to-segment rotations, anchored to real geometry, closed by a cycle constraint. +2.70% accuracy.

Physical intuition

Split a 32-frame clip into three temporal segments — beginning, middle, end. Pool the encoder's per-point features inside each segment into a single descriptor: $f_1, f_2, f_3$. Now ask the model to predict three quaternions: $q_{12}$ rotating segment 1 to segment 2, $q_{23}$ for 2→3, $q_{31}$ for 3→1.

If those three predicted quaternions were unconstrained, the network could trivially output identities and the loss would be zero. To prevent that, we tie them down with two physical constraints:

  1. Reconstruction grounding — applying $q_{12}$ to segment 1's centered XYZ point cloud must actually rotate it onto segment 2's. This is a hard geometric truth measured against raw coordinates, not features. The network can't fake it.
  2. Cycle closure — composing $q_{12} \cdot q_{23} \cdot q_{31}$ must equal the identity quaternion. Going around the loop in time should bring you back to where you started, with no net rotation.

The loss is the sum of three reconstruction MSEs and one cycle-closure MSE. Both terms hold the predicted quaternions in tension: reconstruction ties them to the data, cycle ties them to each other.

Math

$$\textbf{Pool:}\quad f_k \;=\; \mathrm{mean}\big(\,\text{enc}\big(\text{seg}_k\big)\,\big)\;,\qquad k \in \{1,2,3\}$$ $$\textbf{Predict:}\quad q_{12} \;=\; \mathrm{Norm}\!\big(\mathrm{MLP}\big([f_1\,\Vert\,f_2]\big)\big)\;,\qquad q_{23},\;q_{31}\;\text{similarly}$$ $$\textbf{Recon}\quad \mathcal{L}_{\text{recon}} \;=\; \frac{1}{3}\!\!\sum_{(s,t,q)\in\{(1,2,q_{12}),(2,3,q_{23}),(3,1,q_{31})\}}\!\!\!\!\!\!\!\!\!\big\|\;q\,\circ\,(\mathbf{X}_s - \bar{\mathbf{X}}_s)\;-\;(\mathbf{X}_t - \bar{\mathbf{X}}_t)\big\|^2$$ $$\textbf{Cycle}\quad \mathcal{L}_{\text{cycle}} \;=\; \min\!\Big(\,\big\|\,q_{12}\,q_{23}\,q_{31} - e\,\big\|^2,\;\;\big\|\,q_{12}\,q_{23}\,q_{31} + e\,\big\|^2\,\Big)$$ $$\boxed{\;\mathcal{L}_{\text{QCC}} \;=\; \mathcal{L}_{\text{recon}} \;+\; \mathcal{L}_{\text{cycle}}\;}$$

Here $q\circ v$ denotes quaternion rotation of vector $v$, $e = (1,0,0,0)$ is the identity quaternion, and the $\min$ in $\mathcal{L}_{\text{cycle}}$ accounts for the double cover ($q$ and $-q$ represent the same rotation). The reconstruction term operates on raw centered XYZ — never on features — so the predicted quaternion can't escape into a degenerate feature subspace.

Figure 3. Three temporal segments, each pooled into a single feature vector. An MLP predicts the rotation between each pair. Reconstruction loss ties each $q$ to the actual XYZ rotation; cycle loss enforces that going around the loop gives the identity.

Implementation

# reqnn_motion.py · _GroundedCycleConsistency.forward
seg1, seg2, seg3 = pool_per_segment(encoded, num_frames=3)

q12 = norm(quat_head(cat(seg1, seg2.detach())))
q23 = norm(quat_head(cat(seg2, seg3.detach())))
q31 = norm(quat_head(cat(seg3, seg1.detach())))

# Reconstruction: each q must rotate src XYZ onto tgt XYZ
recon = 0
for src, tgt, q in [(xyz1, xyz2, q12), (xyz2, xyz3, q23), (xyz3, xyz1, q31)]:
    src_c = src - src.mean(1, keepdim=True)
    tgt_c = tgt - tgt.mean(1, keepdim=True)
    recon += F.mse_loss(quat_rotate(q, src_c), tgt_c.detach())

# Cycle: composed product must equal identity quaternion
q_cycle = hamilton(hamilton(q12, q23), q31)
cycle   = torch.min(((q_cycle - q_id)**2).sum(-1),
                    ((q_cycle + q_id)**2).sum(-1)).mean()

return recon / 3 + cycle
Why this one works when others didn't

Pooled, not per-point. The loss operates on a single feature vector per segment, not on per-point features. This means classification-relevant variance across points is preserved — the loss can't collapse them.

Hard geometric grounding. The reconstruction term references raw XYZ coordinates. The predicted quaternion has to actually transform geometry; it can't degenerate to a feature-space trick.

Novel constraint. The cycle closure $q_{12}q_{23}q_{31} = e$ is a constraint the classifier loss alone never imposes. It forces the network to learn temporally self-consistent rotation predictions, which appears to translate into more sample-invariant features — exactly the property random TTA exploits for another +1.45%.

The pattern
  • Grounded cycle is the active ingredient — it accounts for the entire +2.70% improvement from no-aux to baseline. Bearing-quaternion rigidity, correspondence sampling, and the various per-point auxiliary losses are scaffolding that doesn't move the needle on its own.
  • The relationship to weight is non-monotonic. $w=0$ → 77.59%, $w=0.1$ → 80.29%, $w=0.2$ → 76.35%. There is a sweet spot where the loss provides structure without overwhelming the classification gradient.
  • Random TTA is QCC-specific. It gives +1.45% on the grounded_cycle model but -0.20% on the no-aux model. The cycle constraint trains features that are more sample-invariant, and TTA harvests that property. On the no-aux model there's nothing to harvest.
  • For fusion, deterministic eval beats TTA. Quat(TTA) is individually higher but its softmax is sharper, which over-weighs it in the linear blend with PMamba. Quat(det) fuses better and gives the new peak: 90.87% with α = 0.8.
