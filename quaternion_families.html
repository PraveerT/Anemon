PMamba quaternion branch

Experiment families

Every quaternion run is identified by a recursive compact name. A family is its depth-0 model code: any chain that roots at the same depth-0 token string belongs to the same family, regardless of what happens at deeper levels.

What is a family

Definition

A family is defined by its depth-0 model code. Two runs belong to the same family if and only if their depth-0 root has the same compact name. The family identifier IS the depth-0 token string.

Example: v6p, v6r, v6s all share family Sc_Co_pr01_N3_pts48-256_e140 — they're three depths of the same family. v6r is preloaded from v6p, v6s from v6r, but the family code stays fixed at the depth-0 root.

Naming convention

Compact format: <preload>_<corr>_<aux>_<segments>_<pts>_<epochs>

Sc
From scratch (random init)
P[<parent>]E<n>
Preloaded from <parent> at epoch n (recursive)
Nc / Co
no correspondence loaded / correspondence loaded
none / gc<w> / pr<w> / co<w>
aux loss type and weight (e.g. gc01=0.1, gc02=0.2)
pr<w>_gc<w>
Stacked aux losses (any combination)
-D / -XF
Cycle module variant: deep_mlp / transformer
N<n>
num_cycle_segments (only meaningful for grounded_cycle)
pts<a>-<b> / pts<n>
Dynamic ramp a→b / static n
e<n>
num_epoch

Chain C — the only fully clean depth-0/1/2 family

Family Sc_Co_pr01_N3_pts48-256_e140 is the cleanest evidence we have for the warm-restart mechanism: every parent in the chain is fully traceable, recipe is identical at every depth, peak accuracy gains compound monotonically.

Sc_Co_pr01_N3_pts48-256_e140 76.14% (v6p) └─E135──→ P[Sc_Co_pr01_N3_pts48-256_e140]E135_Co_pr01_N3_pts48-256_e140 79.05% (v6r) +2.91% └─E125──→ P[P[Sc_Co_pr01_N3_pts48-256_e140]E135_Co_pr01_N3_pts48-256_e140]E125_Co_pr01_N3_pts48-256_e140 80.91% (v6s) +1.86%

Cumulative gain: +4.77% in two warm-restart cycles, with the same recipe at every link.

Family-level findings

  • Only one family has full depth-0/1/2 data with a clean lineage: Sc_Co_pr01_N3_pts48-256_e140. This is the strongest evidence for the warm-restart mechanism.
  • From-scratch baselines cluster at 75-77% regardless of aux loss type. The specific aux variant matters very little at depth 0.
  • The 80%+ ceiling requires either (a) a multi-depth warm-restart chain, or (b) a preload from another good model. No single-shot from-scratch run cracks 78%.
  • The unknown-root family (? — quaternion_branch) reaches 80.29% from a lost control rerun. Its depth-2 children (v6c/d/f) all drop by ~1% when restarted from the peak, demonstrating that restarting from a fully-converged checkpoint hurts more than restarting from a mid-training one.
  • Stacking aux losses is statistically tied with single aux at depth 0: v7a (stacked) ~ v6p (single) ~ v6q (no aux), all within 0.5% of each other.
